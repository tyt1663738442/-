# 2026-05-18 工作记录

## Railway 部署配置适配

### 1. 后端端口自适应（`backend/main.py`）
- 读取 `PORT` 环境变量（Railway 自动注入），本地默认 8001
- 启动时打印当前环境和端口
- 支持 `RAILWAY` 环境变量检测部署平台

### 2. 前端环境配置分离
创建两个配置文件：
- **`frontend/.env.development`**：开发环境，配置 `VITE_API_BASE=http://localhost:8001`
- **`frontend/.env.production`**：生产环境（Docker/Railway），留空使用相对路径

### 3. Vite 配置升级（`vite.config.ts`）
- 使用 `loadEnv()` 动态加载环境变量
- 开发环境：proxy 指向 `VITE_API_BASE`
- 生产环境：`VITE_API_BASE` 为空，不配置 proxy，API 使用相对路径 `/api`

### 4. 新增 `.gitignore`
- 忽略 `.env.local` 和 `.env.*.local`
- 忽略 `node_modules/`、`dist/`、日志文件等

### 5. Dockerfile 注释优化
- 标注 Railway 会自动覆盖 `$PORT`
- 后端 `main.py` 已读取 `$PORT` 环境变量
